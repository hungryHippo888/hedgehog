Hedgehog Iterative Design

Hedgehog, is designing AI-driven robots to optimize the mushroom farming process. Even though parts of the process will be automated, there will still be a need for overview and intervention from experienced mushroom farmers, and the mobile app we have designed is intended for those users to be able to overview the status of the farm, plan out planting and harvesting dates, and interact with the robotics. We hope this app has a positive impact on the workstyles of the users, making their farming work more efficient and less tedious by delegating away automatable tasks, while also letting them work professionally with an app that is intuitive to learn and use daily.

Part 1: Sketching and Wireframing

Sketching

Combining Ideas into a Wireframe

We thought of many different ways to represent the analytics dashboard of all the different clusters in the app, and in the end we chose a spatial representation in the form of a simplified map of the farm. This page acts as the landing page for the app because it functions as the highest level overview of the mushroom farm. It allows the user to see the status of the farm, including any issues with planted mushroom clusters (these issues are marked with a warning icon, implying there needs to be human interaction with an un-automatable task). From this page, the user can also click on empty cluster spaces to show that they will plant a new cluster, or see more information about an already planted cluster by clicking on its box and seeing the details & analytics page.

We decided to create an “Add New Cluster” page to showcase the user adding a new cluster that the robot can interact with. The necessary fields we had were Nickname, Cluster Number, Date Planted, and Mushroom Type. The Nickname, Cluster Number, and Mushroom Type fields allow the user to identify specific clusters, and the Date Planted field records when the user planted a particular cluster of mushrooms.

We created a page for each mushroom cluster that shows the details and analytics of that cluster, including the nickname, cluster number, type, date planted, age, time until harvest, temperature, humidity, and any warnings the cluster may have. The temperature and humidity information comes from the robot’s sensors and are provided to the user to help with any warnings/changes they may need to make. These pages can be accessed by clicking on any mushroom cluster from the home/map page, or by clicking on a cluster’s name from the calendar page.

We decided to build an Apple-inspired calendar page for the user to keep track of dates for planting and harvesting mushroom clusters. We thought that having all of the information on the initial month-calendar page would be overwhelming, so we adopted Apple’s use of dots below the date to signify an event (planting or harvesting) on that day.

If the user clicks on a day, they are taken to the week-calendar page which shows the events of that day, separated into planting and harvesting sections. From these day pages, users can also click on any cluster name to see the details & analytics page!

Feedback from User Testing

Summary of task-specfic feedback

  • landing page tasks - no user errors; most users understand how the design of the map was a digital representation of the physical farm
  • crop status tasks - no user errors; users also noted it was very obvious with the use of the orange warning icon which clusters needed urgent attention
  • calendar tasks - no user errors; however one point that came up more than once was the misuse and inconsistency of the colors, as users assumed green referred to days with a planting event and brown referred to days with a harvesting event (which wasn’t intentional) but since it wasn’t consistent, it confused multiple users

Summary of general feedback

  • users praised the simplicity of the design
  • users loved the colors (with the exception of the calendar inconsistency)
  • users noted the icons were intuitive
  • one user was confused about lack of customization options

    Potential interface changes based on feedback

  • actually implementing the meaning users assumed the colors had (green = planting, brown = harvesting) consistently throughout the whole app
  • including a tutorial for first time users explaining the map (i.e. automatically opening the help icon) and a help pop-up explaining the calendar
  • customization options for the number of clusters and layout of the farm
  • reworking the calendar to: manually add events to calendar (not from plus sign in map screen), weekly viewing mode vs. monthly viewing mode options, should say “today” in the notes when the day being viewed isn’t today
